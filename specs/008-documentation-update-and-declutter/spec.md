# 008: Documentation Update and Declutter

## Purpose
Reshape the documentation set so the repository leads with clear user-facing guidance, while contributor material and command reference content remain easy to find and maintain.

## Seed Scope
- Update README.md so it is user-facing rather than contributor-heavy.
- Plan and write a user manual in human-readable sections, including HTML and TUI-friendly versions.
- Write a `man stack` command manual for the CLI.
- Make sure all commands are easily discoverable.
- Declutter the docs set and organize content into user and developer/contributor sections.

## Expected Outputs
- A revised README focused on installation, first use, and common workflows.
- A documentation plan for the user manual and its output formats.
- A command manual strategy covering CLI discoverability and maintenance.
- A proposed information architecture for user docs versus contributor docs.

## Open Questions
- Should the manual track the `stage` rename immediately, or wait for rename completion?
- Which documentation formats are generated from one source versus maintained separately?
- Where should contributor workflows live after the declutter pass?
- How will command discoverability be tested and kept current?

## Out Of Scope For This Placeholder

- Historical archive cleanup unless it directly blocks active documentation.
- Final copy editing across every doc before the spec is approved.
